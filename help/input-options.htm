Winfingerprint supports four different input options:
  • IP Range
  • IP List
  • Single Host
  • Neighborhood

IP Range

Winfingerprint will scan from the starting IP address through the ending IP address with the selected scanning options, skipping IP address ending in .255. IP Ranges can also be defined using a netmask (Example: 255.255.255.0) or and inverted netmask (Example: 0.0.0.255)

IP List

The full path of a list of IP addresses or NetBIOS names can by typed into the "IP Address List Filename:" edit box or the filename can be chosen via the familiar "Open File" dialog method by pressing the "Browse" button.
The list of IP Addresses/NetBIOS names can be in one of the following formats:
On host per line with no leading spaces.
Example:

10.0.0.1
TEST-W2k
TEST-XP
TEXT-NT4
10.0.0.5
A list of IP Ranges.
Example:
192.168.1.1 - 192.168.1.15
10.0.0.2 - 10.0.0.1.183
A list of IP Ranges using Netmasks.
Example:
192.168.1.1 - 255.255.255.0
10.0.0.2 - 255.255.255.252
A list of IP Ranges using Inverted Netmasks.
Example:
192.168.1.1 - 0.255.255.255
10.0.0.2 - 252.255.255.255

Single Host

Enter the IP Address or NetBIOS name of the target computer and it will be scanned with the selected options.

Neighborhood

Leave this option blank to enumerate the "Network Neighborhood" from the root, or specify a starting domain from which to begin enumeration. NOTE: In Windows domains and workgroups one computer maintains a list of resources on the network. This list, called the Browse List, contains a list of shares, printers, etc, that are available on the network. By default, all computers with the SMB protocol installed advertise themselves to this resource list, maintained by the Master Browser, regardless of whether they have any resources to provide or not.
When "Neighborhood" is selected, this is the list that is scanned. If a computer has set:

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\lanmanserver\parameters]
"Hidden"=dword:00000001
the computer will not be scanned. Return to top of page
