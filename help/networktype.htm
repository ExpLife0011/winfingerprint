Winfingerprint can enumerate OS Version, Users, Groups and Services via three different methods:
NT DOMAIN uses the Net* Win32 APIs to gather information.
Win Fingerprint Domain
Users Domain
Groups Domain
Services Domain

Active Directory uses the ADSI APIs to gather information.
You may have to download the Microsoft Platform SDK for the ADSI method to work although the Winfingerprint installer does include activeds.dll and adsldpc.dll.
Win Fingerprint Active Directory
Sessions Active Directory
* Users Active Directory
* Groups Active Directory
* Services Active Directory
NOTE: Under Windows 2000, The Active Directoy Scan method only works for WinFingerprint and Sessions, and does not work with Users, Groups, and Services. Under Windows XP, WinFingerprint, Users, Groups, Sessions, and Services all enumerate correctly.

WMI uses the Windows Management Instrumentation APIs to gather information.
NOTE: WMI queries against remote systems require administrator privileges.
Winfingerprint's WMI can currently query, OS, Service Pack, Hot Fixes, MAC address, and Running Services.
