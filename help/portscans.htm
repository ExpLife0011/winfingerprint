Winfingerprint can perform TCP SYN, TCP connect (banner grabbing), UDP, and ICMP Scans.
  • TCP Banner Grabbing Portscan
  • UDP Portscan
  • ICMP Scan
Choose a valid port range 1-65535 for both TCP and UDP Portscans (they do not have to be equal).
NOTE: By altering the Timeout (Default 5 seconds), you can adjust the select() timeout for the TCP/UDP/ICMP/SNMP Scans.
The interface drop down selection is only used by the TCP SYN Portscan option to determine which interface (IP address and MAC address) use as the source protocol and hardware address. Max Connections, limits the total number of maximum connections the TCP and UDP connect() scans will attempt during one pass.
TCP SYN and connect() based Portscans
Winfingerprint first performs a TCP SYN scan. If the port is open (responds with a SYN|ACK) and the report typically returns a banner (21/ftp, 22/ssh, 23/telnet, 25/SMTP, 80/HTTP, 110/POP3, 119/NNTP, 143/IMAP, 443/HTTPS), winfingerprint will then perform a TCP non-blocking connect portscan against those open ports.
